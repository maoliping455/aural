# Aural

Aural 是一款本地优先的 macOS 音频/视频转写 App。

它在本机完成音视频导入、本地转写、播放核对和结果导出，默认不需要账号，不上传你的音视频或转写文本。当前 0.1.0 初版面向 Apple Silicon Mac 和 macOS 14 或更新版本。

如果你认可 Aural 的方向，欢迎给项目点一个 GitHub Star。Star 会帮助更多需要本地转写的人看到这个项目，也会帮助我判断后续投入优先级。

## 它解决什么问题

- 音视频隐私：访谈、会议、课程、播客素材、个人笔记和研究资料不一定适合先上传到云端。Aural 默认在本机处理，减少隐私暴露。
- 免费、无云端转码成本：本地模型在你的 Mac 上运行，不按音频分钟数计费，也不需要把视频上传到云端转码后再处理。
- 准确率接近云端水准：当前本地 ASR 链路以 Qwen3-ASR 为核心，在中文、中英混合和长音频场景中追求接近云端服务的可用准确率。
- 可核对、可导出：转写结果可以跟随播放核对，并导出为字幕或文本，方便继续整理、剪辑、写笔记或交给后续 AI 工作流。

## 当前已支持

- 音频导入：`mp3`、`m4a`、`wav`、`aac`、`flac`
- 视频导入：`mp4`、`mov`、`m4v`，导入后提取音频副本用于转写
- 本地转写队列：批量导入后自动排队处理
- 首次模型准备：按提示下载本地转写资源，后续复用本机缓存
- 三种本地转写模式：极速、平衡、精准
- 字幕时间戳对齐：可开启以提升播放定位和字幕导出效果
- 播放核对：拖动进度、跳转时间点、跟随播放位置、段落高亮
- 任务管理：停止、重新开始、删除、重命名和搜索
- 导出结果：字幕 SRT、纯文字 TXT、带分段时间 TXT
- 隐私默认：无账号、无遥测、无云端转写

## 快速开始

1. 从 [GitHub Releases](https://github.com/maoliping455/aural/releases) 下载公开 DMG。
2. 打开 DMG，把 `Aural.app` 拖到 Applications。
3. 首次启动时按提示准备本地模型资源。
4. 导入音频或视频，等待本地转写完成。
5. 播放核对结果，并导出字幕或文本。

0.1.0 采用轻量安装包：App 内置本地运行环境，但不把大模型权重打进 DMG。模型默认保存在：

```text
~/Library/Application Support/Aural/Models
```

如果资源已存在，Aural 会直接复用；如果资源缺失，会在首次使用前提示准备。模型下载需要联网，默认优先使用 ModelScope，必要时 fallback 到 Hugging Face；下载支持断点续传。

## 隐私与本地数据

Aural 不会上传你的音视频和转写文本。

导入文件后，Aural 会在自己的本地目录中创建音频副本；删除任务时，会删除应用内记录、音频副本、转写结果和相关 sidecar 文件，但不会删除你的原始文件。

首次准备本地模型时需要联网下载模型资源；这一步只下载模型，不上传用户媒体或转写文本。

如果你把导出的转写文本交给外部 AI 工具处理，隐私边界取决于那个工具。Aural 只承诺自己负责的媒体处理和本地转写链路不默认上传。

详细说明见 [隐私说明](docs/privacy.md)。

## 长期愿景

Aural 的长期定位不是“大而全 AI 知识库”，而是音视频进入 AI 工作流和长期知识系统之前的本地可信入口。

长期希望做到：

- 把本地音视频转成可追溯、可回放、可引用的结构化资料。
- 保留来源、时间戳、模型版本和处理记录，让转写结果有清晰事实来源。
- 通过稳定的 artifact 和 CLI，让不同 Agent、本地 LLM、脚本和知识系统都能消费。
- 支持把精选内容进入 Markdown、Obsidian 或其他长期知识库。
