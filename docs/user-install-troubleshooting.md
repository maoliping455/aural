# Aural 安装与故障排查

更新时间：2026-07-08

本文档面向普通用户，说明如何安装、首次准备本地模型、处理常见启动/下载/磁盘问题，以及如何卸载 Aural。

## 适用范围

Aural 0.1.0 面向：

- Apple Silicon Mac。
- macOS 14 或更新版本。
- 能接受首次下载本地模型资源的用户。

暂不支持：

- Intel Mac。
- 低于 macOS 14 的系统。
- 纯 CPU 本地转写后端。
- 实时麦克风转写。

## 安装

1. 从 GitHub Releases 下载 Aural 的 DMG。
2. 打开 DMG。
3. 把 `Aural.app` 拖到 `Applications`。
4. 从 Finder 的 Applications 中打开 Aural。

GitHub Releases 中公开下载的 v0.1.0 DMG 应在上传前完成 Developer ID 签名、Apple notarization、staple 和 Gatekeeper 验证。正常情况下，拖到 Applications 后可以直接打开。

如果你使用的是本地开发包、内部临时包，或 macOS 仍提示无法验证开发者：

1. 在 Finder 中找到 `Aural.app`。
2. 右键点击 `Aural.app`。
3. 选择“打开”。
4. 在弹窗中再次确认“打开”。

不要从 DMG 里直接长期运行 App。建议先拖到 Applications 再打开。

## 首次模型准备

Aural 的 DMG 不内置大模型权重。首次启动时，Aural 会检查本机是否已有本地转写资源。

如果资源缺失，Aural 会进入准备页。用户需要选择：

- 本地转写模式：
  - 极速：下载更小，速度更快，质量相对低。
  - 平衡：默认推荐，兼顾速度和准确性。
  - 精准：下载更大、占用更高，需要 16GB 及以上内存。
- 是否启用字幕时间戳对齐：
  - 推荐开启，播放定位和字幕导出更准确。
  - 关闭后下载更少，资源准备更快。

模型默认保存在：

```text
~/Library/Application Support/Aural/Models
```

下载完成后，后续 App 升级会复用本地模型缓存，不需要每次重新下载。

## 常见问题

### 打不开 App

可能原因：

- 下载的不是公开 notarized release 包，或 Gatekeeper 仍在拦截。
- 当前设备不是 Apple Silicon Mac。
- 当前系统低于 macOS 14。

处理方式：

- 先确认下载的是 GitHub Release 中的正式 DMG。
- 如果是本地开发包或内部临时包，按“右键打开”流程操作。
- 如果是设备或系统不支持，Aural 0.1.0 暂时无法继续使用本地转写。

### 一直停在准备模型

可能原因：

- 网络连接不稳定。
- ModelScope 或 Hugging Face 下载较慢。
- 磁盘空间不足。
- 下载被中断。

处理方式：

- 保持网络连接后重试。
- 确认磁盘空间足够。
- 关闭 Aural 后重新打开，再点击重试。
- 已下载的部分会保留，重试时应继续复用。

如果一直失败，可以删除不完整的模型目录后重新准备。删除前请先退出 Aural。

模型目录：

```text
~/Library/Application Support/Aural/Models
```

### 下载占用空间太大

模型资源会保存在本机。大致体积取决于用户选择的模式和是否启用时间戳对齐：

- 极速模式最小。
- 平衡模式是默认推荐。
- 精准模式最大。
- 字幕时间戳对齐会额外下载对齐模型。

如果磁盘空间紧张，可以在本地转写设置中选择更小的模式，或关闭字幕时间戳对齐。已经下载过但不再使用的模型可以手动清理。

清理模型前请先退出 Aural。删除模型后，下次使用对应模式时需要重新下载。

### 不能导入文件

可能原因：

- 本地转写资源还没有准备完成。
- 文件格式不支持。
- 视频没有音轨。

当前支持：

- 音频：`mp3`、`m4a`、`wav`、`aac`、`flac`
- 视频：`mp4`、`mov`、`m4v`

视频导入后，Aural 只保留提取出的音频副本，不会把原始视频复制进 App 数据目录。

### 转写失败

可能原因：

- 音频文件损坏或无法被系统解码。
- 模型资源不完整。
- 本地 runtime 加载失败。
- 转写进程超时或异常退出。

处理方式：

- 重新开始该任务。
- 尝试换一个较短的音频文件验证是否是单个文件问题。
- 确认模型资源已准备完成。
- 如果问题稳定复现，请反馈文件类型、系统版本、Mac 型号、任务状态和错误出现步骤。

不要在公开 Issue 中上传私人音视频或完整转写内容。

### 转写结果有错字、重复或时间戳不准

0.1.0 是本地转写首发版本，ASR 质量仍会受音质、口音、背景噪声、音乐、人声重叠和模型能力影响。

建议处理方式：

- 对重要内容，使用播放核对功能人工确认。
- 对字幕导出，优先开启字幕时间戳对齐。
- 如果出现明显大段重复、幻听或整段漏转，请作为 Bug 反馈。

反馈时建议提供：

- 是否是音频或视频。
- 语言和大致场景。
- 是否开启字幕时间戳对齐。
- 大概出问题的时间点。
- 可公开分享的最小样例。如果素材私密，只描述现象即可。

### 删除任务会不会删除原始文件

不会。

删除任务只会删除：

- Aural 内部任务记录。
- Aural 创建的音频副本。
- 转写结果。
- 时间戳对齐文件。
- 错误日志。

用户原始导入文件不会被 Aural 删除。

## 卸载

如果只想删除 App：

1. 退出 Aural。
2. 删除 Applications 中的 `Aural.app`。

如果还想删除本地任务、转写结果和模型缓存，请同时删除：

```text
~/Library/Application Support/Aural
```

删除该目录会移除 Aural 的本地任务、转写结果、模型缓存和设置。之后重新安装或启动 Aural 时，需要重新准备模型资源。

## 反馈问题

反馈 Bug 时，建议包含：

- Aural 版本。
- macOS 版本。
- Mac 型号和内存。
- 文件类型和大致时长。
- 问题发生步骤。
- 是否首次启动、是否刚下载模型、是否开启字幕时间戳对齐。
- 是否能稳定复现。

请不要公开上传私人音视频、完整转写文本、含个人信息的日志或账号凭据。
